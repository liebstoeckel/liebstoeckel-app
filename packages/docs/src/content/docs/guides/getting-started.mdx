## Prerequisites

- **[Bun](https://bun.com) 1.3+** — liebstoeckel uses Bun as runtime, bundler, and test runner.
- A browser. (Optional: a Chromium for build-time thumbnails — see [Thumbnails](/guides/thumbnails/).)

<Aside type="note" title="Pre-release">
The framework isn't on the public npm registry yet, so the `bunx liebstoeckel …`
forms below are how it will work once published. Today, run the CLI from a checkout
of the repo (`bun run …`) or a workspace that links the packages.
</Aside>

## Scaffold a deck

The umbrella CLI scaffolds a workspace-wired deck:

<Steps>

1. Create a deck:

   ```bash
   liebstoeckel new my-talk            # or: bunx liebstoeckel new my-talk --brand acme
   ```

2. Install dependencies (links the workspace packages):

   ```bash
   bun install
   ```

3. Start the dev server (instant HMR + React Fast Refresh):

   ```bash
   liebstoeckel live my-talk           # or, from the deck dir: bun run dev
   ```

</Steps>

A new deck looks like this:

<FileTree>
- my-talk/
  - index.html        the page shell (sets `data-brand`)
  - main.tsx          mounts `<Present>` with your slides
  - build.ts          single-file build (+ thumbnails)
  - server.ts         HMR dev server
  - bunfig.toml        dev-server plugins (tailwind + mdx)
  - slides/
    - 01-intro.tsx
</FileTree>

## The entry point

`main.tsx` wires a brand, plugins, and ordered slides into the engine:

```tsx title="main.tsx"
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { Present } from "@liebstoeckel/engine";
import "@liebstoeckel/theme/styles.css";

import Intro from "./slides/01-intro";

createRoot(document.getElementById("root")!).render(
  <StrictMode>
    <Present title="My Talk" brands={["nocturne"]} slides={[Intro]} />
  </StrictMode>,
);
```

<Aside type="tip">
`brands` is an array — pass more than one (e.g. `["acme", "sunset"]`) and press <kbd>t</kbd> while presenting to cycle the whole deck through them.
</Aside>

## Build a single file

```bash
liebstoeckel build my-talk            # → my-talk/dist/my-talk.html
```

The output is one self-contained `.html` (named after the deck — `dist/<deck-slug>.html`) (JS/CSS inlined, assets base64'd, slide thumbnails embedded). Open it directly, email it, or host it anywhere static.

<Aside type="note" title="Bun-only build API">
Decks are bundled with the **`Bun.build()` JS API** (in `build.ts`), not the `bun build` CLI — `compile: true` + `target: "browser"` is what inlines everything. See [Building & deploying](/guides/building/).
</Aside>

## Keyboard during a talk

| Key | Action |
| --- | --- |
| <kbd>→</kbd> / <kbd>Space</kbd> | next reveal, then next slide |
| <kbd>←</kbd> | previous |
| <kbd>o</kbd> | overview grid |
| <kbd>p</kbd> | open presenter window |
| <kbd>f</kbd> | fullscreen · <kbd>b</kbd> blur screen |
| <kbd>q</kbd> | show the join QR (live) · <kbd>t</kbd> cycle brand |
| <kbd>123…</kbd>+<kbd>Enter</kbd> | jump to slide · <kbd>?</kbd> help |
