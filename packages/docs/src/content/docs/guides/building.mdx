## One self-contained file

A deck builds to a single `dist/index.html` with JS and CSS inlined and assets base64'd. There's nothing else to host.

```bash
present-it build my-talk        # → presentations/my-talk/dist/index.html
```

Under the hood, `build.ts` calls the engine's build API (which wraps `Bun.build`):

```ts title="build.ts"
import { buildDeckWithThumbnails } from "@present-it/thumbnails/build";

// builds the single file, then embeds slide thumbnails (skips gracefully
// if no Chromium / when PRESENT_IT_NO_THUMBS=1)
await buildDeckWithThumbnails({ entry: "./index.html", outdir: "./dist" });
```

<Aside type="note" title="Why the JS API, not the CLI">
The single-file output requires the **`Bun.build()` JS API** with `compile: true` + `target: "browser"` — that's what inlines everything. The `bun build` *CLI* does not produce the same standalone file. The engine plugins (Tailwind, MDX, the code macro, Shiki) run in this build.
</Aside>

## What ends up in the file

- Your bundled, minified JS (engine + slides + plugins' client code).
- The compiled CSS (Tailwind + brand variables + code/atmosphere styles).
- Pre-tokenized highlighted code (no Shiki at runtime).
- The **plugin manifest** — a JSON `<script>` listing the deck's plugins, with any server bundles base64-encoded (decoded + run only by a live server, never in the browser).
- **Slide thumbnails** — a manifest of WebP data-URIs for the overview grid (see below).

## Deploying

Because it's one static file, deploy anywhere:

- Open `dist/index.html` directly (file URL) — it just works offline.
- Drop it on any static host (S3, GitHub Pages, Netlify, a plain web server).
- Email it / put it in a shared drive.

To present it **live** to a room or the internet, you don't deploy the file — you run [`present-it live`](/guides/live/) (optionally [via a relay](/guides/relay/)).

## Opting out of thumbnails

Thumbnails need a headless Chromium. The build **skips them gracefully** (with a note, never failing) when none is available or when you set:

```bash
PRESENT_IT_NO_THUMBS=1 present-it build my-talk
```
