## Why liebstoeckel

liebstoeckel is a small, opinionated **engine + theming layer**, not another reveal.js: content is authored in MDX/TSX, design lives in one place per brand, and motion is first-class.

<dl class="home-ledger">
  <div class="lrow">
    <dt>Single-file output</dt>
    <dd>Every deck builds to one portable <code>.html</code>: JS and CSS inlined, assets base64'd. No server, no CDN. It drops into an email, a USB stick, an air-gapped room. And it isn't a dead end: <code>liebstoeckel eject</code> turns a built file back into an editable project.</dd>
    <a class="lrow-link" href="/guides/building/">guides/building <span aria-hidden="true">&rarr;</span></a>
  </div>
  <div class="lrow">
    <dt>Motion that matters</dt>
    <dd>Magic Move (<code>layoutId</code>), spring choreography, animated code diffs: the reason to build our own engine instead of bending a generic tool.</dd>
    <a class="lrow-link" href="/guides/animated-code/">guides/animated-code <span aria-hidden="true">&rarr;</span></a>
  </div>
  <div class="lrow">
    <dt>Brands, not styles</dt>
    <dd>A deck declares <code>brand: "acme"</code> and inherits the whole look from typed design tokens. Override one token for a one-off without forking the theme.</dd>
    <a class="lrow-link" href="/guides/theming/">guides/theming <span aria-hidden="true">&rarr;</span></a>
  </div>
  <div class="lrow">
    <dt>Decks that talk back</dt>
    <dd>Host the same file and it opens a live Yjs session: polls, audience Q&A, and reactions sync to every device in the room. Unhosted, plugins render a quiet fallback.</dd>
    <a class="lrow-link" href="/guides/live/">guides/live <span aria-hidden="true">&rarr;</span></a>
  </div>
  <div class="lrow">
    <dt>Built for agents</dt>
    <dd><code>liebstoeckel skill install</code> teaches your agent the same CLI you use: it reads the component registry, writes MDX and TSX you own, and loops on <code>build --check</code> until the deck is green.</dd>
    <a class="lrow-link" href="/reference/cli/">reference/cli <span aria-hidden="true">&rarr;</span></a>
  </div>
  <div class="lrow">
    <dt>Mobile-ready</dt>
    <dd>The audience joins from a phone. Tap a glowing plugin to interact full-size, swipe to navigate, and the presenter view adapts down to a handset.</dd>
    <a class="lrow-link" href="/guides/mobile/">guides/mobile <span aria-hidden="true">&rarr;</span></a>
  </div>
</dl>

<p class="home-note">These docs are agent-readable too: every page has <strong>Copy Markdown</strong> and <strong>Open in ChatGPT / Claude</strong>, a raw <code>.md</code> at each URL, and an <a href="/llms.txt">/llms.txt</a> index.</p>

## The 30-second tour

```bash frame="none"
bun add -g @liebstoeckel/cli       # or: bunx liebstoeckel …
liebstoeckel new my-talk           # scaffold a deck
liebstoeckel live my-talk          # present live (audience joins by QR)
liebstoeckel build my-talk         # → one self-contained dist/index.html
```
